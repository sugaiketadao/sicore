Page Layout.

  • <header> and <footer> are fixed at the top and bottom.
  • <main> uses all remaining height except <header> and <footer>.
  • <section> provides page left/right margins and section top/bottom margins. Do not use <section> to remove top/bottom margins.
[Example]
<body>
  <header>
    <section>Fixed Header</section>
  </header>
  <main>
    <section>Main Content</section>
  </main>
  <footer>
    <section>Fixed Footer</section>
  </footer>
</body>

Grid Layout (12-Column).

  • Uses a layout method that divides the page width into 12 equal parts for flexible element placement.
  • Row elements are specified with div.grid-row, with column widths specified per row.
  • Column elements are specified with div.grid-col-1 to div.grid-col-12, where the trailing number represents the column width.
  • Column width uses values from 1 to 12, with the total equaling 12.
  • Column width is normally 1/12 of the page, but on smartphone sizes it becomes full width and columns stack vertically.
  • Content within columns is normally arranged horizontally, but on tablet sizes it changes to vertical arrangement.
[Example 1]
<div class="grid-row">
  <div class="grid-col-3">1/4 of full width</div>
  <div class="grid-col-3">1/4 of full width</div>
  <div class="grid-col-3">1/4 of full width</div>
  <div class="grid-col-3">1/4 of full width</div>
</div>
[Example 2] Widths do not need to be uniform
<div class="grid-row">
  <div class="grid-col-6">1/2 of full width</div>
  <div class="grid-col-4">1/3 of full width</div>
  <div class="grid-col-2">1/6 of full width</div>
</div>

Form Input Element Styles.

  • Unifies styles for form input elements such as text boxes, select boxes, and text areas.
  • Unifies styles for normal, focus, and disabled states.
  • Unifies styles for labels paired with form input elements.
  • Wrap label elements with div.item-head and form input elements with div.item-body to apply height adjustment and responsive direction switching styles. 
    [Example]
<div class="item-head"><label>Item Name</label></div>
 <div class="item-body"><input type="text"></div>
  • Wrap checkboxes and radio buttons with their value labels in <label>. 
    [Example]<div class="item-body"><label><input type="checkbox">Value Name</label></div>

Table Styles.

  • Unify the style of <table>.
  • Tables display with alternating row stripes.
  • Wrap <table> with div.table to apply styles. 
    [Example]
<div class="table"><table>...omitted...</table></div>
  • Wrap <th> with <thead> and <tr> with <tbody> to apply styles.
  • <tbody> can be single or multiple. 
    [Single Example] <table>...omitted...<tbody><tr></tr><tr></tr><tr></tr><tr></tr></tbody></table>
[Multiple Example] <table>...omitted...<tbody><tr></tr><tr></tr></tbody><tbody><tr></tr><tr></tr></tbody></table>

Utility Styles.

  • This section provides commonly used general-purpose styles.
  • Use the following classes for text alignment:
    • Left align: .align-left
    • Center align: .align-center
    • Right align: .align-right
  • Set .danger on elements requiring caution to display in warning color (used for delete buttons, etc.).
  • Set .button on <a> to display it with button-like appearance.

Message Display / Item Highlight Styles.

  • The HTML for the message display area uses the following fixed format:
    <section id="_msg"></section>
  • Calling JavaScript framework component PageUtil#setMsg() sets the following classes on target item elements, highlighting those items:
    • Info (blue background): .info-item
    • Warning (yellow background): .warn-item
    • Error (red background): .err-item
  • Items to highlight are specified along with message text in the argument of Java framework component Io#putMsg().
